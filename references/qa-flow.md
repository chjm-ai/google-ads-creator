# 任务一：生成投放计划 — 问答流程 + 生成规则

通过结构化问答收集业务信息，再由 AI 生成完整 Google Ads 投放策略文档。

## 问答流程

**分轮询问，每轮不超过 3-5 个问题。** 不要一次抛出全部问题，根据用户回答动态调整。

### 第一轮：业务基本面
1. 你的公司/品牌叫什么？主营什么产品或服务？
2. 你的目标客户是谁？（B2B 还是 B2C？什么行业？什么角色会搜索你？）
3. 你的核心竞争优势是什么？（价格、品质、速度、定制、地理位置…列 2-3 个）

### 第二轮：投放目标与预算
4. 投放目标是什么？（询盘/电话、网站流量、品牌曝光、App 下载…）
5. 每日预算大概多少？（给货币单位）
6. 试水还是已有经验？（决定策略激进程度）

### 第三轮：市场与竞争
7. 想投哪些国家/地区？有没有要排除的？
8. 落地页/网站 URL 是什么？（没有也可，后续补）

可选（按上下文判断）：
9. 客户通常搜什么词找到你？
10. 知道竞争对手投什么词吗？
11. 网站上有 WhatsApp / 在线聊天 / 询盘表单吗？
12. 有投放时段偏好吗？（如只工作日）

### 第四轮：多语言与技术细节（如适用）
13. 网站支持哪些语言？（判断是否需要多语言广告组 + 对应着陆页）
14. 目标国买家主要用什么语言搜索？（如印尼买家用印尼语搜 `pemasok PVC`）
15. 产品价格区间 / 单笔订单价值（判断出价策略和 ROI 预期）
16. 是否有线下展厅 / 可验厂（决定是否加地标词组）
17. 主要沟通方式（WhatsApp / Email / 电话）
18. 是否有产品认证（REACH、RoHS、ISO 等，影响文案和信任度）

## 网站结构分析（用户提供 URL 后自动执行）

提供 URL 后，**生成计划前**必须先爬取分析网站结构，为计划和 CSV 的着陆页 URL / Sitelink / Path 提供真实数据，不用占位符。

### 分析流程
1. **爬取首页**：用 `defuddle` 或 `WebFetch` 取首页，提取导航菜单结构
2. **爬取关键子页面**：产品/服务页、产品分类页、关于我们、联系/询盘页；有多语言则访问各语言版本
3. **记录 URL 清单**：整理可用于广告的 URL 列表

### 需要提取的信息
| 信息 | 用途 |
|------|------|
| 产品分类 URL | Ad Group 对应着陆页（`/products/pvc-resin/`） |
| 多语言 URL 结构 | 本地语言 Ad Group 着陆页（`/id/products/`） |
| 联系/询盘页 URL | Sitelink Extension |
| About/资质页 URL | Sitelink Extension |
| 是否有 WhatsApp 按钮 | 决定是否设 Call Extension + 转化追踪 |
| 是否有询盘表单 | 转化追踪设置 |
| 网站支持的语言 | 多语言策略 |
| 产品/服务列表 | 关键词生成依据 |

### 分析输出：URL 映射表
整理成表后直接嵌入计划和 CSV：

```markdown
### 网站 URL 映射（自动生成）
| 用途 | URL | 语言 | 备注 |
|------|-----|------|------|
| 主着陆页 | https://example.com/products/ | EN | 产品总览 |
| PVC 原料着陆页 | https://example.com/products/pvc-resin/ | EN | Campaign 1 |
| 印尼语产品页 | https://example.com/id/products/ | ID | 印尼语 Ad Group |
| 联系/询盘页 | https://example.com/contact/ | EN | Sitelink + 转化 |
| WhatsApp | wa.me/8613800138000 | — | Call Extension |
```

URL 真实性见根目录 SKILL.md 的 Gotchas 段。

## 计划生成规则

收集足够信息（问答 + 网站分析）后，按 `templates/campaign_plan_template.md` 的 11 章节结构输出，每章须有实质内容：

1. 执行摘要 2. 目标市场分析（各国 CPC 预估/买家特征/语言） 3. 广告账户结构（Campaign 划分、国家级出价调整） 4. 关键词策略（核心词+长尾+本地语言+否定词） 5. 广告文案（A/B 版本、Extensions） 6. 网站分析与着陆页策略（URL 清单、Ad Group→着陆页对应、Sitelink） 7. 转化跟踪（GTM 事件清单） 8. 预算分配与出价（投放时段、设备出价） 9. 投放时间表（基建→优化→扩量） 10. 核心 KPI 与预期 11. 上线前 Checklist

### AI 生成时的专业规则
1. **关键词**：基于行业+产品生成，每组 6-10 个英文词，从买家搜索视角出发（不是卖家自嗨词）
2. **广告文案**：严格遵守字符限制（Headline ≤30、Description ≤90），逐条验证
3. **市场选择**：低预算（<$30/天）集中 1-2 个市场
4. **出价策略**：低预算 Manual CPC；中高预算 Maximize Clicks 或 Target CPA
5. **否定词**：必含通用否定词（retail, shop, amazon, job, pdf, free 等），再按行业补
6. **广告组划分**：按搜索意图分组（产品词/地标词/OEM 定制词/品牌词），不按产品分
7. **多语言**：网站多语言时为主力市场加本地语言 Ad Group，着陆页对应到对应语言版本
8. **Campaign 结构**：
   - <$1000/月：按产品线分 Campaign，国家用 Location Bid Adjustment
   - $1000-5000/月：按国家分 Campaign，产品线用 Ad Group
   - >$5000/月：国家 × 产品线矩阵，每组合独立 Campaign
9. **投放时段**：B2B 默认周一至周五当地 08:00-18:00，说明各目标国与中国时差
10. **设备出价**：初始不调整，跑 2 周后按转化数据调（移动差则 -20%，桌面好则 +15%）
11. **着陆页对应**：每个 Ad Group 明确对应着陆页 URL，不能全落首页，URL 须来自网站分析确认存在的真实页面
12. **Ad Extensions**：必含 Sitelink、Callout、Structured Snippets；有 WhatsApp 加 Call Extension
13. **A/B 文案**：每个广告组至少 2 套版本
14. **预算分配表**：含月预算、日预算、预估 CPC、预估月点击量

## 计划输出
- 路径：`~/Desktop/GoogleAds_[品牌名]_YYYYMMDD/投放计划.md`
- 对话中展示摘要让用户确认/修改
- 用户确认后，询问是否继续任务二生成导入文件
